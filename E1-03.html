Exam 1 - 03 - Permutation and Combination

1. Overall Learnings / Summary of the Topic

Permutation

 nPr = n! / (n - r)!
- Number of ways objects can be rearranged or ordered
- Order matters
Where: n is the total number of objects or elements in the set. r is the number of objects chosen or arranged from the set. n! denotes the factorial of n, which is the product of all positive integers up to n.

Combination

 nCr = n! / (r! * (n - r)!)
- Number of ways objects can be chosen or selected
- Order does not matter
Where: n is the total number of objects or elements in the set. r is the number of objects chosen or selected from the set. n! denotes the factorial of n, which is the product of all positive integers up to n.

2. Analysis

Permutation and combination are some of the fundamental concepts in discrete math. These concepts deal with counting the number of ways objects can be rearranged or chosen, in order or not in order. This is the most used concept in computer science, especially in algorithm designing, data structures, optimizations, and more.

3. Applications, Real-Life Scenarios or Examples

Used in:
- Algorithms
- Data Structures
- Finding the complexity of choices
- Analyzing probability and randomness
- Optimization

Game Development (Permutation)

Example 1.3.1 - Scenario: You're working on a puzzle game in which players must arrange a series of colored blocks in a precise order. Each block can be colored in one of three ways: red, green, or blue. The goal is to determine the number of possible arrangements or permutations of the blocks.

Solution:
We must determine the number of permutations of a set of blocks with three potential colors (red, green, and blue) in this challenge. Because the order is important, we can compute the permutations using the formula nPr = n!, where n represents the number of objects.

Because we have three blocks to arrange in this situation, the number of permutations is:
3P3 = 3! = 3 x 2 x 1 = 6

Therefore, there are 6 possible arrangements or permutations of the colored blocks in the game.

Image Processing (Combination)

Example 1.3.2 - Scenario: Assume you're working on an image processing algorithm that adds a filter to a picture. The filter has various changeable parameters, and you wish to test different parameter combinations to obtain the optimum image-enhancing outcomes.

Solution:
In this situation, we must determine the number of parameter value options for the image processing algorithm. Assume we have three modifiable parameters, each of which can have four different values.

The formula nCr = n! is used to calculate the number of combinations. / (r! * (n - r)!), where n is the total number of objects and r denotes the number of objects chosen.

In this case, we have 4 values for each of the 3 parameters, so the number of combinations is:
3C3 = 3! / (3! * (3 - 3)!) = 3! / (3! * 0!) = 1
Therefore, there is only 1 combination of parameter values for the image processing algorithm. This means that the algorithm will be applied with all possible combinations of parameter values, resulting in a single enhanced image.

4. Reflection and Thoughts

Permutation and combination are important concepts in discrete math that analyze the possibilities and ways objects or choices could be chosen or rearranged. It is so far the easiest yet trickiest topic in this course as the concepts in a problem could be mixed, making it challenging to solve. Later on in the next parts of exam 2, permutation and combination are used almost exclusively for every concept there. This topic was aided by a different course from computer science, Elementary Statistics, where the same two concepts are used to find the probability of order and choosing. Understanding these concepts solidified my understanding of the next portfolio topics.
