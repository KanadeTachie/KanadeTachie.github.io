Exam 1 - 02 - Pigeonhole Principle

1. Overall Learnings / Summary of the Topic

The principle came from the analogy of pigeons and pigeonholes where there are more pigeons than there are more pigeonholes. So, at least one of the pigeonholes accommodates multiple pigeons at once.

If ‘n’ items are distributed into ‘m’ boxes or containers where ‘n’ is greater than ‘m’, then at least 1 box or container must contain more than 1 item.
OR
If k+1 objects are assigned to k places, then at least 1 place must be assigned >= 2 objects.
Generalized pigeonhole principle
If n objects are placed in k containers, then at least one container has ceiling [ n/k ] objects
Note: x is the smallest integer greater than or equal to x. When x = 3.0001, it becomes 4

2. Analysis

Also known as the Dirichlet drawer principle or the box principle, it is another fundamental concept in combinatorics. The pigeonhole principle is one of the main principles frequently used in searching algorithms. The rules of this principle are used to understand and analyze the time complexity and behavior of the algorithms. The fact that one container contains at least more than one of something means this principle could be used in detecting repetitions of something. Time complexity analysis, repetition detection, and data storage index are many of the main applications of this principle. Understanding this principle is key in designing algorithms and analyzing the time it takes for them to finish, their best cases and worst cases, and behaviors.

Worst Case Probability

Example 1.2.1 - Socks : Suppose you have six socks of three different colors: two red, two blue, and two white. If you randomly select three socks, according to the pigeonhole principle, there must be at least one color that has two socks among the selected three. This is because there are more socks (6) than the number of colors (3).

Duplicate Searching Algorithm

Example 1.2.2 - Finding Duplicates in an Unsorted Array : Given an unsorted array of integers, design an algorithm to determine if there are any duplicate elements present in the array. Output true if there are duplicate elements, false if there are none.

Problem Solution


This is an example problem that utilizes the pigeonhole principle. Since there are more elements in the array (n) than there are the number of distinct elements, there must be at least one element that is a duplicate. Distinct elements here are the boxes or ‘pigeonholes’ and the pigeons would be the number of elements.
